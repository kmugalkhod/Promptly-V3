## When to Use
- Selecting fonts for a new app design
- Matching typography to chosen aesthetic
- Determining type scale (tight/normal/loose)

## Instructions

### FONT PAIRING OPTIONS

Choose a pairing that matches your aesthetic:

| Pairing | Display Font | Body Font | Best For |
|---------|--------------|-----------|----------|
| editorial | Playfair Display | Source Serif 4 | Blogs, news, portfolios |
| brutalist | Space Mono | Work Sans | Creative, experimental |
| playful | Fredoka | Nunito | Kids apps, casual |
| luxury | Cormorant Garamond | Montserrat | Finance, premium |
| retro | Righteous | Poppins | Games, tech demos |
| geometric | Outfit | Inter | Construction, industrial |
| humanist | Fraunces | Source Sans 3 | Health, wellness |
| minimal | DM Sans | DM Sans | Productivity, dev tools |

### TYPOGRAPHY SCALE

- **tight**: Compact spacing, dense information display
- **normal**: Balanced, general-purpose
- **loose**: Spacious, editorial feel, luxury apps

### CRITICAL RULES
1. NEVER use Inter as the default font - it's generic and overused
2. ALWAYS pair a display font (headings) with a body font (paragraphs)
3. Match the pairing to the aesthetic for cohesive design
4. All fonts must be Google Fonts (available for free)
